## Technical Claim Rules

- Describe Hydrogen as Shopify's custom storefront toolkit for React Router apps, optimized for Storefront API data.
- Describe Oxygen as Shopify's hosting/deployment layer for Hydrogen storefronts at the edge.
- Describe the Hydrogen channel as the path that enables Oxygen hosting, storefront management, environment variables, deployment logs, and related deployment setup.
- Describe headless Shopify as broader than Hydrogen: Hydrogen is one opinionated Shopify stack, Hydrogen React can support React-based frameworks, and Storefront API/Headless channel can support other custom storefront approaches.
- Use SEO language around route metadata, canonical URLs, JSON-LD, sitemap, robots, rendered HTML, and product-state consistency.
- Avoid older "Remix-only" wording unless referring to legacy examples or older Hydrogen versions; current buyer-facing content should prefer React Router language.

## Buyer Interpretation

Official Shopify facts should not be pasted as generic documentation summaries. Each page should explain what the platform fact means for a merchant decision: whether to audit, migrate, build, optimize, support, stay on Liquid, or delay a rebuild.
